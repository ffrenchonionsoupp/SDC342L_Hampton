# Complaint Management System

## Project Description
Complaint Management System is a PHP and MySQL web application designed to allow customers to submit complaints, technicians to manage assigned complaints, and administrators to oversee complaint assignments and user management. The application includes complaint tracking, technician notes, status updates, and file uploads.

---

## Project Tasks

### Task 1: Set up the development environment
- Install XAMPP and configure Apache/MySQL
- Create the project folder structure and database connection

### Task 2: Design the database
- Create tables for users, complaints, notes, assignments, and roles
- Configure foreign key relationships between tables

### Task 3: Develop authentication
- Implement login/logout functionality
- Add role-based access for customers, technicians, and administrators

### Task 4: Develop customer functionality
- Allow customers to submit complaints
- Add file upload support
- Allow customers to view and update complaints

### Task 5: Develop technician functionality
- Allow technicians to view assigned complaints
- Add technician notes and status updates

### Task 6: Develop administrator functionality
- Allow administrators to assign complaints to technicians
- View all complaints and assignments

### Task 7: Test the application
- Test database functionality and page navigation
- Debug file upload and MVC path issues

### Task 8: Document the project
- Create testing and retrospective documents
- Create README documentation

## Notes

- Import the provided SQL file into phpMyAdmin before testing.
- Uploaded files are stored in the `uploads` folder.
- User passwords are stored using password hashing.

## Running the Project

1. Clone or download the project from the GitHub repository
2. Place the project folder inside the `htdocs` directory in XAMPP
3. Start Apache and MySQL in XAMPP
4. Import the included SQL file into phpMyAdmin
5. Open the project in the browser:

```bash
https://localhost/finalweekproject
```
